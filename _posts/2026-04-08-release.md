### Product Enhancement :

#### Users Can Target Specific Columns in Remove Duplicates –

Users can now target specific columns in the Remove Duplicates rule, deduplicating rows based on a chosen subset of fields rather than the entire row.

#### Pipeline Step Reordering by Handle –

Pipeline steps can now be reordered using a dedicated drag handle on each card, giving users more precise control over the transformation sequence.

#### Redesigned Resource Finder –

A redesigned resource finder now replaces the previous dataset picker across Join, Lookup, Branch Out, Group Pivot, and other features, providing a faster and more consistent selection experience.

#### Project Roles & Permissions –

Users can now see their project role — Admin or Editor — directly in the project switcher, with a Roles & Permissions view detailing what each role allows across project settings, user management, and data access. The app enforces these boundaries in-app: Editors cannot create new data connections and see a clear prompt to request Admin access.


### Defect Fixes :
Resolved the following issues:

* Dashboard — Clicking a dashboard from the left navigation displayed an error, preventing it from loading.
* Dashboard — After publishing a view as a dashboard, the pipeline card incorrectly showed "last updated by deactivated user" instead of the active user's name.
* Pipeline — Adding a task to a pipeline in an error state showed a continuous loader without completing.
* Pipeline — A newly created empty pipeline incorrectly displayed two plus-sign buttons at once.
* Pipeline — The Notes display toggle had no effect on the pipeline card.
* Pipeline — Switching from Large to Compact card view caused the accordion toggle to disappear and the add button to misalign.
* Pipeline — Discarding a task in Draft Mode did not apply correctly; toggling Draft Mode off and back on was required to recover.
* Pipeline — Column names renamed in the source data view did not update in the Label rule panel until the rule was reopened.
* Pivot — The Pivot step card did not display the configured output columns in the pipeline view.
* Join — Renaming a source data view column caused it to be automatically deselected when reopening a Join rule.
* Mini Pipeline — Reordered pipeline steps showed no visual indicator of their new position in the mini pipeline view.
* Branch Out — The dataset search in the result dataset selection modal was not functioning correctly.
* Dataflow Monitor — Searching by action name in the Active Tasks tab returned no results for tasks with Auto Sync off.
* Data Quality — Clicking "View data quality" had no effect.
* Data Check — The "Notify on failure" option did not require an email address before the configuration could be saved.
* Messaging Orchestration — Views linked to an orchestration were not visually highlighted when editing the orchestration.
* Filters — Conditional filters did not apply correctly in the app.
* Group & Pivot — After applying a date format to a column, the data type shown in view metadata differed from what the pipeline step displayed.
* Export — Switching an existing pipeline export from one external database destination to another resulted in a failed update; the destination type is now correctly locked once configured.
